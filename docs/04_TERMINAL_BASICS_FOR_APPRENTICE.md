# Terminal Basics for the Apprentice Engineer  
###### <C:/dev/repos/dead-box-to-dev-box/docs/04_TERMINAL_BASICS_FOR_APPRENTICE.md>

This document teaches the apprentice how to operate the Ubuntu Terminal.  
These skills are required before learning core commands, VS Code, or the compiler.

Follow each step exactly.

---

# 1. Open the Terminal

Press:

```
Super
```

Type:

```
Terminal
```

Press **ENTER**.

Operator rule:

- The Terminal is the apprentice’s primary tool.  
- It must always be opened this way.

---

# 2. The Prompt

A typical prompt looks like:

```
benny@machine:~$
```

Meaning:

- **benny** → username  
- **machine** → computer name  
- **~** → home directory  
- **$** → ready for a command  

Operator rule:

- The apprentice does not modify the prompt.

---

# 3. Typing Commands

Commands are typed exactly and executed with:

```
ENTER
```

Example:

```
ls
```

Press **ENTER**.

---

# 4. Clear the Screen

Press:

```
Ctrl + L
```

This clears the Terminal without deleting history.

Operator rule:

- The apprentice uses **Ctrl + L**, not `clear`.

---

# 5. Cancel a Command

Press:

```
Ctrl + C
```

This stops the current command.

Operator rule:

- The apprentice uses **Ctrl + C** whenever something is stuck.

---

# 6. Command History

Press:

```
Up Arrow
```

to repeat previous commands.

Press:

```
Down Arrow
```

to move forward in history.

Operator rule:

- The apprentice uses history to avoid retyping.

---

# 7. Autocomplete

Type the first few letters of a command or filename, then press:

```
Tab
```

If there is one match, it completes automatically.  
If there are multiple matches, press **Tab** twice to list them.

Operator rule:

- The apprentice uses autocomplete constantly.

---

# 8. Paths

A path is a location in the file system.

Examples:

```
~
/home/benny
/home/benny/projects
```

Operator rule:

- The apprentice must recognize `~` as “home”.

---

# 9. Running Programs

To run a program in the current directory:

```
./program-name
```

Example:

```
./main
```

Operator rule:

- The apprentice must include `./` when running local executables.

---

# 10. Completion

When the apprentice can:

- open the Terminal  
- type commands  
- use ENTER doctrine  
- clear the screen  
- cancel commands  
- use history  
- use autocomplete  
- understand paths  
- run executables  

…they have the required Terminal foundation for all future operator work.
